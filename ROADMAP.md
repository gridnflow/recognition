# 개선 로드맵

향후 세션에서 이 앱을 발전시킬 때의 가이드. 위에서부터 우선순위 순이며, 각 항목에 구현 힌트와 완료 기준을 적어 뒀다. 착수 전 CLAUDE.md의 아키텍처/환경 함정을 먼저 읽을 것.

---

## P1. 게임 체감 다듬기 (작은 작업, 효과 큼)

### 1-1. 판정 메시지에 이유 표시
- **문제**: 게임오버 시 "묵찌빠! 승리!!"/"졌다..."만 떠서 *왜* 끝났는지(같은 손) 안 보임.
- **구현**: `main.App._apply_result()`의 GAME_OVER 분기에서 `"같은 손! 공격자 승리"` 식으로 양쪽 손과 이유를 메시지에 포함. `HudState`에 필드 추가 없이 `message`만 고치면 됨.
- **완료 기준**: 게임오버 화면만 보고 규칙을 모르는 사람도 승패 이유를 이해할 수 있다.

### 1-2. 타이밍 튜닝 옵션
- **문제**: `BEAT`(0.55s), `CAPTURE_SECS`(0.7s)가 하드코딩. 사람마다 반응 속도가 다름.
- **구현**: `argparse`로 `--beat`, `--capture` CLI 옵션 추가. 게임 중 `+`/`-` 키로 실시간 조절해도 좋음.
- **완료 기준**: 코드 수정 없이 구호 속도를 바꿀 수 있다.

### 1-3. 캡처 실패 UX 개선
- **문제**: 손 인식 실패 시 "다시!"만 나옴. 왜 실패했는지(손 없음 vs 모양 애매) 구분 안 됨.
- **구현**: `GestureBuffer`에 "손은 보였지만 애매" 카운트 추가(`add()`에서 None 이전에 landmarks 유무를 받도록 시그니처 확장). 실패 사유별 메시지 분기: "손이 안 보여요" / "손 모양이 애매해요 — 확실하게 내주세요".
- **완료 기준**: 실패 원인별로 다른 안내가 나온다.

## P2. 심리전 컴퓨터 AI

- **현재**: `main.App._finish_capture()`에서 `random.choice(list(Hand))`.
- **구현**:
  - `ai.py` 신설. 플레이어의 손 이력을 기록하고 **1차 마르코프 전이 행렬**(직전 손 → 다음 손 빈도)로 다음 손을 예측.
  - 공격일 때: 플레이어가 낼 것 같은 손과 **같은 손**을 낸다 (같으면 승리이므로).
  - 수비일 때: 플레이어 예측 손과 **다른 손** 중 가위바위보로 이기는 손 (공격권 탈환).
  - 난이도: `--level easy|normal|hard` — easy는 순수 랜덤, normal은 70% 예측/30% 랜덤, hard는 100% 예측.
  - `game.py`는 건드리지 말 것 (순수 로직 유지). AI는 main에서 주입.
- **테스트**: `test_ai.py` — 고정 이력을 넣고 예측이 최빈 전이를 따르는지, 공격/수비 시 손 선택 규칙이 맞는지.
- **완료 기준**: hard 모드에서 같은 패턴만 내면 눈에 띄게 지기 시작한다.

## P3. 2인 대전 모드

- **구현**:
  - `HandLandmarker` 옵션 `num_hands=2`로. `HandReader.read()`가 손 목록을 반환하도록 확장 (기존 1인 코드 경로 호환 유지).
  - 화면 좌/우 절반으로 플레이어 배정: 각 손의 손목 x좌표 < 0.5 → P1, ≥ 0.5 → P2. 거울 반전 후 좌표 기준임에 주의.
  - `game.py`는 이미 두 손 입력을 받으므로 **변경 불필요** — `PLAYER`/`COMPUTER` 상수 이름만 `P1`/`P2` 별칭 고려.
  - `main.py`에 모드 선택 화면 (START 상태에서 1/2 키), 2인 모드에선 캡처 시 양쪽 버퍼 각각 다수결.
  - UI: 컴퓨터 패널 대신 양쪽 하단에 각자 인식 결과 표시.
- **함정**: 두 사람 손이 화면 중앙에서 겹치면 배정이 흔들림 → 캡처 윈도우 동안 손별 배정을 고정(첫 프레임 기준)할 것.
- **완료 기준**: 두 명이 한 웹캠 앞에서 한 게임을 완주할 수 있다.

## P4. 인식 품질 개선

- **엄지 판정 추가**: 현재 `gesture.FINGERS`는 엄지 제외. 보(빠)와 주먹(묵) 구분엔 영향 없지만, 엄지만 편 주먹이 가끔 애매 판정됨. 엄지는 tip(4)이 검지 MCP(5)에서 멀어진 정도로 판정하면 안정적.
- **히스테리시스**: 실시간 표시("나: 찌")가 프레임마다 깜빡일 수 있음 → 직전 판정과 다르면 2~3프레임 연속일 때만 갱신.
- **분류 디버그 모드**: `--debug` 플래그로 손가락별 펴짐 비율(dist(tip)/dist(pip))을 화면에 표시 — 임계값 1.15 튜닝용.
- **완료 기준**: 일반 조명에서 묵/찌/빠 오인식이 체감상 사라진다.

## P5. 연출 / 사운드

- **효과음**: 승리/패배/공격권 이동에 `afplay /System/Library/Sounds/*.aiff` (Glass=승리, Basso=패배 등). `sound.py`에 `play_effect(name)` 추가, `say`와 병행 가능.
- **카운트다운 애니메이션**: 구호 글자를 박자 시작에 크게(140pt) → 끝에 원래 크기로 보간. `ui.render()`에 `center_scale` 필드 추가.
- **연승 표시**: `App`에 스코어 외 연승 카운터, 3연승부터 "🔥 3연승!" 배너.
- **하이스코어 저장**: `scores.json`에 누적 전적 저장/로드 (`.gitignore`에 추가할 것).

## P6. 배포 (.app 패키징)

- `pyinstaller --windowed`로 .app 번들 생성. `models/hand_landmarker.task`와 폰트 경로를 `--add-data`로 포함하고, 코드의 절대 폰트 경로(`ui.KR_FONT` 등)는 번들 내 상대 경로 폴백 추가.
- **TCC 주의 (중요)**: ad-hoc 서명(`codesign --sign -`)은 재빌드마다 카메라 권한이 무효화되어 매번 다시 물어봄. 자체 서명 인증서 또는 Apple Development 인증서로 서명해 정체성을 고정할 것. 꼬였으면 `tccutil reset Camera <번들ID>` 후 한 번만 재허용. 권한 허용 후 **앱 재시작 필요**.
- **완료 기준**: 더블클릭으로 실행되는 .app이 재빌드 후에도 카메라 권한을 유지한다.

## 작업 방식 (모든 항목 공통)

1. 규칙/로직 변경은 `game.py`(순수 로직) + 테스트 먼저, UI/타이밍은 `main.py`/`ui.py`에서.
2. 변경 후 CLAUDE.md의 검증 절차 3단계(단위 테스트 → 헤드리스 스모크 → 실플레이)를 반드시 수행.
3. 한 항목 = 한 커밋(영어 메시지), 완료 기준을 커밋 본문에 확인 형태로 남길 것.
